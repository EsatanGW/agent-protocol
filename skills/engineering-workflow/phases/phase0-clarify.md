# Phase 0: Requirement Clarification

## Goal

Without writing production code, establish clarity on:
- The requirement itself.
- Change boundaries.
- Affected surfaces.
- Blockers.
- Public behavior impact.

## Lean / Full minimums

### Lean minimum
- Affected surfaces.
- Main flow summary.
- Public behavior impact.
- Open questions (if any).

### Full minimum
- Complete draft spec.
- Acceptance-criteria draft.
- Blockers / assumptions / boundaries.

## Built-in steps

1. Read the requirement source.
2. Do a quick scan first, then ask questions.
3. Mark affected surfaces.
4. Judge public behavior impact.
5. Form the question list.
6. Produce the output matching the mode:
   - Lean: minimal clarification note.
   - Full: draft spec.

## Gate 0

The phase passes only when:
- Blockers are either resolved or explicitly acknowledged.
- Affected surfaces are fully marked.
- Public behavior impact has been judged.
- There is enough artifact to support entry into the next phase.
