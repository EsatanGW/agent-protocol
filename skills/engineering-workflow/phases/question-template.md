# Phase 0 Question Template

## Standard questions

1. Which surfaces does this change touch?
2. Who will see or consume this capability?
3. Where is the source of truth?
4. Are we adding any new fields, states, or enums?
5. What are their types, ranges, and default values?
6. How do we handle legacy data and legacy consumers?
7. Does this require coordination with other people, external systems, or manual steps?
8. Once created, is it mutable?
9. Does this need to appear in list views / reports / external lookups?
10. Is there a launch deadline or change freeze?
11. Are there related tickets / RFCs / design documents we should read together?

## Ticket-specific questions

After the standard questions, add the task-specific blockers and clarifications.
