# Phase 8 Trigger Guide

Consider adding a post-delivery observation note when:
- public behavior changed
- a migration was executed
- financial / auth / high-risk flows changed
- staged rollout or feature flags were involved
- the team will need T+24h / T+72h / T+7d observation

Not every task needs Phase 8.
But if the real-world outcome matters more than merge-time correctness, mention it explicitly in delivery.
